# 反制推荐模块 — 开发规则与算法原理

## 一、模块概述

**功能定位**：在命定花种页面，当用户选择一只花种精灵（Boss）后，自动分析该精灵的攻击画像，并推荐最适合应对（反制）它的精灵 Top 20。

**控制方式**：通过管理端 `site_settings` 表的 `counter_picks_enabled` 开关控制是否在用户端展示。

---

| 层级 | 文件 | 职责 |

## 三、算法流程（5大步骤）

### Step 1：确定目标精灵的攻击倾向

```
输入：精灵uid + 可选性格覆盖(natureOverride)
1. 获取精灵的 atk/matk
2. 如果有性格覆盖，应用 ±10% 修正
3. 判定攻击倾向：atk >= matk → 物攻，否则 → 魔攻
4. 确定防御维度：物攻倾向 → 候选精灵看 def；魔攻倾向 → 看 mdef
```

### Step 2：收集攻击属性

```
优先级：
1. 如果该精灵有命定花种技能配置(fate_flower_skills)：
   - 精灵本体属性 + 命定花种配置的攻击技能属性
2. 如果没有配置：
   - 精灵本体属性 + 所有自身攻击技能的属性（fallback）

同时检测：
- hasStatusSkills：是否有状态技能
- hasDefenseSkills：是否有防御技能
```

### Step 2b：分析Boss的状态效果

```
从Boss的技能描述和特性描述中检测关键词：
- "灼烧" → 火系(id=3)
- "寄生" → 草系(id=2)
- "中毒" → 毒系(id=10)
- "冰冻/冻结" → 冰系(id=7)

用途1：排除主属性依赖该状态的候选精灵（因为Boss免疫该状态）
用途2：bossHasBurnOrPoison = 是否有灼烧/中毒 → 触发净化和落陨星兔加成
```

### Step 3：加载属性克制数据

```
- 加载所有属性的 strong_against / resisted_by / weak_to / resistant_to
- 计算目标精灵的弱点属性(targetWeakTo)：哪些属性克制它，以及克制倍率
  - 克制Boss单属性 → ×2
  - 克制Boss双属性 → ×3（双重克制）
- 计算Boss的抵抗属性(bossResists)：用于评估吸血技能有效性
```

**克制倍率规则**（重要）：
- `targetWeakTo` 为 `Map<elementName, multiplier>`
- 如果攻击属性只克制Boss的一个属性 → multiplier = 2
- 如果攻击属性克制Boss的两个属性（双属性Boss） → multiplier = 3
- 三倍克制的技能在评分时获得 **1.5倍** 加成系数（相对于双倍克制）

### Step 4：获取所有最终形态精灵并去重 + 批量预查询

**去重规则**（同一pet_id的多形态）：
- 六维+属性完全相同 → 再比较攻击技能集
  - 技能集也相同 → 只保留一个
  - 技能集不同 → 都保留
- 六维/属性有差异 → 都保留

**批量预查询**（性能优化，避免N+1查询）：

| 查询目标 | 检测条件 | 对应标签 |
|----------|----------|----------|
| `counterStatusPets` | `description LIKE '%应对状态%'` | ⚡ 应对状态 |
| `counterDefensePets` | `description LIKE '%应对防御%'` | 🛡️ 应对防御 |
| `counterAttackPets` | `description LIKE '%应对攻击%'` | 🔰 应对攻击 |
| `lifestealPetSkills` | `name IN ('蝙蝠','暗突袭','撕裂','等价交换','抽枝','气沉丹田')` | 💚 续航 |
| `greedyPets` | `name = '贪婪'` | 🌟贪 |
| `cleansePets` | `name IN ('除厄','洗礼','生日蛋糕','清洗')` | 🌟净 |
| `superEffectiveSkillsByPet` | `element IN (targetWeakTo) AND power > 0 AND type IN ('物攻','魔攻') AND skill_type != 'bloodline_skills'` | ⚔️ 克制 |

### Step 5：评分与排序

**排除条件**（直接淘汰）：
1. 不抵抗Boss任何攻击属性 → 排除
2. 被Boss任何攻击属性克制 → 排除
3. 主属性依赖Boss使用的状态效果 → 排除

### ⚔️💥 / ⚔️ / 🗡️ — 克制攻击技能（分三档）

| 项目 | 内容 |
|------|------|
| **对应技能** | 所有满足条件的攻击技能：`element IN (targetWeakTo) AND power > 0 AND type IN ('物攻','魔攻') AND skill_type != 'bloodline_skills'` |
| **检测方式** | 属性在Boss弱点列表中的攻击技能 |
| **权重** | W_SE_ATTACK = **6** × 技能评分 × 克制倍率系数 |
| **排除规则** | 能量消耗 > 5 的技能直接跳过不计 |

**克制倍率系数**（重要）：
- ×2 克制（单属性克制）：系数 = 1.0
- ×3 克制（双属性克制）：系数 = 1.5

**三档展示规则**：

| 图标 | 条件 | 含义 |
|------|------|------|
| ⚔️💥×3 | `se_attack_score >= 2.5` + `best_se_multiplier = 3` | 高威力三倍克制（深红加粗标签） |
| ⚔️💥×2 | `se_attack_score >= 2.5` + `best_se_multiplier = 2` | 高威力双倍克制（红色标签） |
| ⚔️×3 | `se_attack_score >= 1.5` + `best_se_multiplier = 3` | 中等威力三倍克制（深红加粗标签） |
| ⚔️×2 | `se_attack_score >= 1.5` + `best_se_multiplier = 2` | 中等威力双倍克制（红色标签） |
| 🗡️×N | `se_attack_score > 0` | 低威力克制（红色标签） |

**视觉区分**：
- ×3 克制：深红背景 `bg-red-200`，加粗字体，更醒目
- ×2 克制：浅红背景 `bg-red-100`，普通字体
- 标签右上角显示 `×2` 或 `×3` 上标数字

**技能评分细则**：
```
有效威力(effPower) = 技能威力 × (对应攻击值 / max(atk, matk))

基础评分等级：
- effPower ≥ 120：有应对=3分，无应对=2分
- effPower ≥ 70：有应对=2.5分，无应对=1.5分
- effPower ≥ 40：有应对=2分，无应对=1分
- effPower > 0：有应对=1分，无应对=0.5分

克制倍率加成：
- 基础评分 × 克制倍率系数（×2=1.0, ×3=1.5）

修正：
- 能量消耗 > 4 且威力 ≥ 70：降级0.5分（在倍率加成之前）
- 第二好的技能按50%权重叠加到总分
- "应对"加成仅在Boss有状态技能时生效
```

## 六、评分优先级总览

```
贪婪(10) = 净化(10，需Boss有灼烧/中毒)
  > 克制Boss的吸血续航(8)
  > 三倍克制大威力应对(6×3×1.5=27，Boss有状态时)
  > 双倍克制大威力应对(6×3×1.0=18，Boss有状态时)
  > 三倍克制大威力(6×2×1.5=18)
  > 双倍克制大威力(6×2×1.0=12)
  > 属性抵抗(3×N)
  > 不被抵抗的吸血续航(2)
  > 应对防御(0.5)
  > 防御数值(0~0.5)
  > 被抵抗的吸血续航(-1)

特殊：落陨星兔(999) → 强制置顶
```

## 七、克制倍率规则（重要）

### 游戏规则
- 攻击属性克制防御方的**一个**属性 → ×2 伤害
- 攻击属性克制防御方的**两个**属性（双属性Boss） → ×3 伤害（非×4）
- 双重抵抗 → ×0.25

### 算法实现
- `targetWeakTo` 数据结构：`Map<elementName, multiplier>`
  - key: 克制Boss的属性名
  - value: 克制倍率（2 或 3）
- 计算方式：遍历所有属性，对Boss的每个属性检查是否被克制
  - 只克制Boss的一个属性 → multiplier = 2
  - 克制Boss的两个属性 → multiplier = 3

### 评分加成
- 克制技能的最终评分 = `基础评分 × 倍率系数`
- 倍率系数：
  - ×2 克制 → 系数 1.0（基准）
  - ×3 克制 → 系数 1.5（显著加成）
- 这确保三倍克制的技能在排序中明显优于双倍克制的同威力技能

### 示例
假设Boss是 火/草 双属性：
- 水系攻击：克制火(×2) + 克制草(×2) → 实际×3 → 系数1.5
- 冰系攻击：只克制草(×2) → 实际×2 → 系数1.0
- 水系120威力技能评分：2分 × 1.5 = 3分
- 冰系120威力技能评分：2分 × 1.0 = 2分

## 八、技能弹窗筛选规则

点击推荐精灵后弹窗展示的技能，经过 `filterRelevantSkills()` 筛选，只保留以下类型：

| 筛选条件 | 对应标签 | 说明 |
|----------|----------|------|
| 攻击技能 + 属性克制Boss | `⚔️ ×N克制`（红色，N=2或3） | 非血脉技能才算，×3深红加粗 |
| 描述含"应对攻击" | `🔰 应对攻击`（绿色） | 防御型反制 |
| 描述含"应对状态" + Boss有状态技能 | `⚡ 应对状态`（橙色） | |
| 描述含"应对防御" | `🛡️ 应对防御`（青色） | |
| type = '状态' | `📋 状态`（灰色） | 参考展示 |
| 描述含"驱散自己"+"减益" | `🌟 驱散减益`（靛蓝色） | |
| 技能名在吸血列表中 | `💚 续航`（翠绿色） | 受应对状态条件限制 |

**排序规则**（非血脉Tab）：按有效威力降序
```
有效威力 = 基础威力 × 攻击值比例 × 蓄力惩罚(0.5) × 应对加成(1.2)
```

## 十一、API 接口

**请求**：`GET /api/pets/counter-picks/:uid?nature=性格名`

**响应结构**：
```json
{
```
