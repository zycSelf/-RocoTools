# 公告表格展示规范

> 本文件定义赛季公告（patch_notes / launch_notes）中所有表格的统一展示规则。
> 脚本路径：`scripts/generate_patch_notes.js`、`scripts/generate_launch_notes.js`
> 前端渲染：`app/client/src/views/user/Season.vue` 的 `parseMarkdown()` 函数

---

## 一、通用规则

### 1.1 表格分类与 cols-N 映射

前端 `parseMarkdown()` 会自动为 `<table>` 添加 `class="cols-N"`（N = 列数），不同列数有不同的响应式适配策略：

| 列数 | Class | 适配策略 |
|------|-------|----------|
| 2 | `cols-2` | 第一列 sticky 130px + 第二列 word-break 自动换行 |
| 3 | `cols-3` | `width: max-content; min-width: 100%`，每列 min-width 150px |
| 6 | `cols-6` | 默认 nowrap，第一列 sticky（技能名/精灵名） |
| 8 | `cols-8` | 第一列 sticky + 背景色覆盖 + 阴影分隔 + z-index 分层 |

### 1.2 排序规则

| 数据类型 | 排序规则 |
|----------|----------|
| 精灵列表 | 按 `pet_id` 数字升序 |
| 技能列表 | 按属性顺序（ELEMENT_SORT_ORDER）→ 技能ID（uid 数字部分）升序 |
| 进化组 | 组内按 `pet_id` 升序，组间按首只 `pet_id` 升序 |

### 1.3 图标语法

| 语法 | 渲染尺寸 | 用途 |
|------|----------|------|
| `![pet:uid]` | 24px 圆形 | 精灵名称前缀 |
| `![skill:uid]` | 20px | 技能名称前缀 |
| `![element:path]` | 1.2em (≈20px) | 属性标识 |
| `![ability:path]` | 2em | 特性名称前缀 |
| `![img:path]` | 56px | 缩略图/立绘 |
| `![shiny:uid]` | 56px | 异色展示（加载失败自动隐藏） |

### 1.4 免责说明

每个 section 标题后统一加引用块：

```markdown
> 以下XXX可能包含前赛季遗漏的XXX，已与游戏实际情况比对，如有出入以官方为准。
```

### 1.5 空值处理

- 数值为 null/undefined 时显示 `-`
- 无变化的字段显示 `-`（数值调整表格）或省略
- 描述文本截取前 60 字符，换行替换为空格

## 二、各模块表格规范

### 2.1 更新概览（2列）

```markdown
| 类别 | 数量 |
|------|------|
| 新增精灵 | 25 只 |
| 　传说精灵 | 1 只 |    ← 子项用全角空格缩进
```

### 2.2 精灵展示表格（2列：图鉴+信息）

适用于：传说精灵、通行证精灵、赛季奇遇精灵、赛季奇遇异色精灵

```markdown
| <div style="min-width:130px;display:inline-block">图鉴</div> | <div style="min-width:360px;display:inline-block">精灵信息</div> |
|------|---------|
| ![img:缩略图] ![shiny:uid] | 进化链（属性图标+属性名系）`标签`<br>**特性** ![ability:path] **「特性名」** 描述<br>HP **X** / 速度 **X** / 物攻 **X** / 魔攻 **X** / 物防 **X** / 魔防 **X** 　总计 **X** |
```

**要点**：
- 图鉴列：常规缩略图 + 异色图（同一单元格）
- 信息列：3行用 `<br>` 分隔（名称行 + 特性行 + 数值行）
- 双属性：`![element:path1]属性1 / ![element:path2]属性2`
- 进化组取 total 最高者展示数据

### 2.3 全部新增精灵（3列）

```markdown
| 精灵 | 精灵 | 精灵 |
|------|------|------|
| ![pet:uid] **名称** ![element:path]`标签` | ... | |
```

每行 3 只，不足补空。

### 2.4 新增技能表格（6列）

```markdown
| 技能名 | 属性 | 类型 | 能耗 | 威力 | 描述 |
|--------|------|------|------|------|------|
| ![skill:uid] 技能名 | ![element:path] | 类型 | 能耗 | 威力 | 描述(≤60字) |
```

**要点**：
- 属性列只显示图标，不显示属性文字

### 2.5 精灵数值调整表格（8列）

```markdown
| 精灵 | HP | 速度 | 物攻 | 魔攻 | 物防 | 魔防 | 总计 |
|------|----|----|----|----|----|----|------|
| ![pet:uid] 名称 | 旧→新(±差) | ... |
```

- 有变化：`旧值→新值(+差值)` 或 `旧值→新值(-差值)`
- 无变化：`-`

### 2.6 精灵个体值补充表格（8列）

同 2.5 结构，但直接展示数值（不显示 `0→X` 格式）。

### 2.7 技能学习面变动（6列 + 精灵子行）

```markdown
| 技能 | 属性 | 类型 | 能耗 | 威力 | 效果/精灵 |
|------|------|------|------|------|----------|
| ![skill:uid] **技能名** | ![element:path] | 类型 | 能耗 | 威力 | 描述 |
| | | | | | **精灵技能**：![pet:uid] 名称(LvX)　　![pet:uid] 名称(LvX) |
| | | | | | **血脉技能**：![pet:uid] 名称　　![pet:uid] 名称 |
| | | | | | **技能石**：![pet:uid] 名称　　![pet:uid] 名称 |
```

**要点**：
- 按技能分组（非按精灵分组）
- 属性列只显示图标，不显示属性文字
- 精灵列表每行最多 4 只，用全角空格 `　　` 分隔
- 续行无前缀（直接从精灵图标开始）
- 升级技能附带等级 `(LvX)`，血脉/技能石不附带

### 2.8 技能调整表格（6列）

```markdown
| 技能 | 属性 | 类型 | 能耗 | 威力 | 调整内容 |
|------|------|------|------|------|----------|
| ![skill:uid] **技能名** | ![element:path] | 类型 | 能耗变化 | 威力变化 | 效果变动描述 |
```

**要点**：
- 属性列只显示图标，不显示属性文字
- 能耗/威力列：有变化时显示 `旧→新`，无变化时显示当前值
- 调整内容列：新旧效果分两行展示 `旧效果<br>→ 新效果`
- 仅名称/属性/类型变化时单独列出
- 排序：属性顺序 → 技能ID

### 2.9 精灵特性调整表格（2列：精灵+调整内容）

```markdown
| 精灵 | 特性调整 |
|------|----------|
| ![pet:uid1] ![pet:uid2] ![pet:uid3]<br>![pet:uid4] ![pet:uid5] ![pet:uid6] | ![ability:path] **「特性名」** 旧描述<br>→ 新描述 |
```

**要点**：
- 同一特性描述变动的进化链合并为一行
- 精灵列：显示所有精灵图标，每3个一行用 `<br>` 换行；不显示名称、不显示"等N只"
- 调整内容列：特性图标（1.2em小尺寸）+ `**「特性名」**` + 旧描述 `<br>→` 新描述（分两行）
- 仅特性名变化时：`特性名 旧名 → 新名`
- 排序：按 pet_id 升序

## 三、前端 CSS 适配要点

### 3.1 通用

```css
.table-wrap { overflow-x: auto; }  /* 横向滚动容器 */
table { border-collapse: collapse; font-size: 0.78rem; }
```

### 3.2 Sticky 列规则

- sticky 列需要独立背景色（亮色 `#fff` / 暗色 `#1f2937`），覆盖斑马纹
- 偶数行 sticky 列也需要单独设置背景色
- `th:first-child` z-index: 5 > `td:first-child` z-index: 2

### 3.3 cols-2 特殊处理

- `table-layout: fixed` 强制列宽分配
- 第一列固定宽度 130px，`white-space: nowrap`，`overflow: visible`
- 第二列 `white-space: normal; word-break: break-word; padding-left: 1rem`

### 3.4 cols-6 适配

- 第一列 sticky + 背景色覆盖 + 阴影分隔（与 cols-8 一致）
- 最后一列 `white-space: normal; word-break: break-word; min-width: 200px; max-width: 400px`
- 第4/5列（能耗/威力）`min-width: 45px` 防止表头换行
