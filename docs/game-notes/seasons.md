# 赛季系统

## 赛季周期

- 游戏按赛季更新（S1、S2...）
- 每个赛季有独立的限定内容

## 通行证精灵

- 每赛季推出通行证，含 **2 只**通行证精灵可选
- 获取方式：
  - 自己拥有的果实
  - 去拥有果实的好友世界，通过重复捕捉概率获取

## 传说精灵

- 每赛季有 **1 只**主推的传说精灵
- 当赛季主要获取途径，不排除后续赛季返厂

## 赛季奇遇精灵

- 每赛季有 **8 只**仅当赛季可捕捉的精灵（原称"赛季限定精灵"）
- 这 8 只拥有异色版本
- 赛季结束后，异色版本仅可通过孵蛋获取

## 赛季奇遇异色精灵

- 每赛季有 **8 只**日常可捕捉的精灵，在当赛季可于野外获取其异色版本（原称"赛季异色精灵"）
- 赛季结束后，异色版本仅可通过孵蛋获取
- 精灵本体不受影响，随时可捕捉

## 赛季切换规则

- 赛季结束后，上述 16 只精灵（8 赛季奇遇 + 8 赛季奇遇异色）的异色版本：
  - 野外不再产出
  - 仅可通过**孵蛋**获取

## 数据结构（管理端配置）

| 字段 | 说明 |
|------|------|
| season_id | 赛季标识（如 S1、S2） |
| pass_pets | 通行证精灵（2只 uid，JSON数组） |
| legend_pet | 传说精灵（JSON数组，支持多只，如 `["pet_295","pet_152"]`） |
| season_pets | 赛季奇遇精灵（8只 uid，JSON数组，拥有异色） |
| shiny_pets | 赛季奇遇异色精灵（8只 uid，JSON数组，当赛季野外可获取异色） |
| announcement_url | **赛季详情**公告链接（官方公告地址） |
| announcement_text | **赛季详情**公告横幅文案 |
| announcement_content | **赛季详情**公告正文（Markdown格式，赛季详情页弹窗展示） |
| home_announcement_url | **首页**公告链接 |
| home_announcement_text | **首页**公告横幅文案 |
| home_announcement_content | **首页**公告正文（Markdown格式，首页弹窗展示） |

## 公告系统

赛季公告分为两套独立配置：

| 公告位置 | 字段前缀 | 展示位置 | 说明 |
|----------|----------|----------|------|
| 赛季详情公告 | `announcement_*` | 赛季详情页 | 展示当前赛季相对于上赛季的改动 |
| 首页公告 | `home_announcement_*` | 用户首页 | 展示当前赛季改动或其他形式文本公告 |

- 两套公告独立配置，互不影响
- 赛季切换时自动切换对应公告内容（读取 `is_current=1` 的赛季）
- 公告内容支持 Markdown 格式，含自定义语法（见下方）

## 公告 Markdown 自定义语法

| 语法 | 渲染结果 |
|------|----------|
| `![pet:uid]` | 精灵头像图标（24px圆形） |
| `![skill:uid]` | 技能图标（20px） |
| `![img:path]` | 内联图片（56px，固定尺寸） |
| `![shiny:uid]` | 异色立绘（56px），图片加载失败时自动隐藏"异色：图片"整体 |
