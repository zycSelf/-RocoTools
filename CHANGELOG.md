## V1.0 — 首个完整版本 (2026-05-30)

> 从项目创建到首个功能完整的公开版本，涵盖数据爬取、后台管理、用户端展示全链路。

### 🏗️ 基础架构

- Python 爬虫系统（并发 + 防限流 + 指数退避重试）
- Node.js/Express + SQLite3 后台 API
- Vue3 + Vite + TailwindCSS 前端 SPA
- Nginx (HTTP/2 + Brotli) + PM2 生产部署
- 自动部署：crontab 每日 00:00 拉取 + deploy.sh
- 前端路由挂载到 `/rocotools/` 子路径
- SEO 预渲染（各路由独立 meta/JSON-LD/noscript）
- sitemap.xml + robots.txt
- ICP 备案展示
- 自定义 favicon 图标

### 🎨 前端特性

- **响应式三端适配**：手机/平板/PC，Mobile-first
- **暗色模式**：全站支持 light/dark 切换
- **精灵详情导航**：
  - PC (≥1280px)：热区触发悬浮按钮，鼠标靠近边缘显示
  - 手机/平板 (<1280px)：滑动手势切换，拖拽跟手+回弹效果
  - 首次引导：全屏mask遮罩 + 手指动画，完成一次滑动后消失
  - 边缘30px排除区域，避免与浏览器返回手势冲突
- **图片系统**：懒加载 + 并发队列(max 6) + WebP自动返回
- **文本高亮**：18属性色 + 独立色 + 六维色
- **公告Markdown**：自定义语法 (pet/skill/ability/element/shiny图标)
- **公告表格**：cols-N体系，sticky列 + 横向滚动
- **导航标签**：数据库驱动，支持父子结构二级下拉
- **API缓存**：内存缓存 5-10分钟，写操作自动清除
- **请求超时**：用户端20s/管理端30s/爬取60s，AbortController
- **页面可见性**：5分钟后台切回自动刷新

### 🤖 数据管理

- **sync_db.js 10步同步**：缩略图→WebP→建表→导入→迁移→规范化→清洗→进化链→最终形态→默认课题
- **数据保护**：manual_edit=1 保护 + UPSERT策略 + 完整性校验
- **自动备份**：auto_presync_*.db (最近5份)
- **进化链系统**：二维数组多路线 + 结构化条件4类型 + 自动同步
- **UID稳定化**：_uid_mapping.json 持久化多形态映射
- **远程同步**：sync_from_server.sh (DB/图片/赛季备份)

## 版本管理规范

### 版本号规则

- **V{主版本}.{次版本}**
- 主版本 (Major)：重大里程碑、架构重构、不兼容变更
- 次版本 (Minor)：新功能模块、重要优化

### 发布流程

1. 确认所有功能已完成并测试
2. 更新 `CHANGELOG.md`（添加新版本section）
3. 更新 `.ai-memory.md`（同步最新状态）
4. 更新 `DOC_RULES.md`（整理时间）
5. Git 提交：`git commit -m "release: V{版本号} - {简要描述}"`
6. Git 推送，服务器自动部署

### 记录格式

每个版本包含：
- 版本号 + 发布日期
- 一句话概述
- 按模块分类的变更列表（使用 emoji 分类标识）
