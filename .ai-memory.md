## 核心数据模型

7 层数据：**属性(18) → 技能(469+) → 蛋组(15) → 精灵(466+) → 性格(30) → 赛季 → 活动日历**

| 数据 | UID/PK | 存储 |
|------|--------|------|
| 属性 | `elem_{id}` | data/elements/ |
| 技能 | `skill_{N}` | data/skills/ |
| 蛋组 | 按名称 | data/eggs/ |
| 精灵 | `pet_{id}` 或 `pet_{id}_{N}` | data/pets/ |
| 性格 | id (1-30) | data/natures/ |
| 赛季 | S1/S2... | 管理端配置（仅数据库） |
| 活动 | 自增 id | season_events 表（管理端配置） |

## 爬虫 (crawler/)

- 入口：`crawler/run.py --full` / `--update`
- 防限流：UA 池 + 随机间隔 + 指数退避重试
- 执行顺序：属性 → 技能 → 蛋组 → 性格 → 精灵列表 → 精灵详情
- 完成后自动：打印完整性汇总 + 同步 SQLite 数据库
- 每个爬虫自动生成 `*_report.md` 完整性校验报告

---

## 后台 (app/server/)

- Express + better-sqlite3 + dotenv
- RESTful API：elements/skills/eggs/pets/natures/seasons/events
- 管理端 API：`/api/admin/*`（JWT 鉴权）
- API 内存缓存：elements/eggs/natures/seasons 10分钟，skills/pets/events 5分钟
- 静态资源：`/public/` → data/public/，`/uploads/` → data/uploads/
- 前端托管：server/public/，SPA fallback

### 数据同步 sync_db.js
1. 生成缩略图（128px WebP q60）→ 更新 pet_list.json
2. 建表（CREATE IF NOT EXISTS，不删库）+ **自动迁移缺失列**（migrateColumns）
3. 导入数据（跳过 manual_edit=1 的记录，冲突存入 pending_conflicts.json）

### 数据库迁移
- init.js 的 `migrateColumns()` 自动解析 schema.sql，对比实际表结构，ALTER TABLE ADD COLUMN 补充缺失列
- 无需删库重建，新增字段只需改 schema.sql 后跑 sync_db.js

### 数据保护
- `manual_edit` 字段：管理端编辑自动标记为 1
- import.js 跳过标记记录，生成冲突报告
- 管理端「数据审查」页面逐条对比处理

## 管理端

- 鉴权：JWT（4h有效），密码/secret 通过 .env 配置
- 通用 CRUD：EDITABLE_TABLES 配置表名/主键/可编辑字段/autoIncrement
- 图片上传：IMAGE_TYPES 配置目录/后缀/DB映射，isUpload 类型存到 data/uploads/
- 备份系统：
  - 临时备份（backups/）：1次确认删除
  - 赛季备份（backups/seasons/）：3次确认删除
  - 恢复前快照（backups/snapshots/）：可命名
- 赛季管理：通行证(2)+传说(1)+限定(8)+异色(8)，PetPicker 选取
- 活动管理：版本活动(banner+单段日期)+常驻课题(多段日期)，支持图片上传
- 通用 CRUD create API 返回 lastInsertRowid（供图片上传关联用）

## 重要约定

1. `data/` 不入 git（含图片和生成数据）
2. `data/uploads/` 不入 git（手动上传图片，不被爬虫覆盖）
3. `deploy.sh`、`setup-nginx.sh`、`DEPLOY.md`、`.env` 不入 git
4. `nginx.conf` 入 git 但用占位符（`<YOUR_DOMAIN>`/`<PROJECT_DIR>`）
5. init.js 不删除旧数据库（保留 seasons 等手动录入数据）+ 自动迁移缺失列
6. 管理端编辑自动标记 manual_edit=1，import 时跳过
7. 赛季/活动数据仅在管理端配置，不通过爬虫获取
8. 活动 API 默认只返回当前活跃的（end_date >= today），管理端传 all=1 获取全部
9. 通用 CRUD 的 autoIncrement 表创建时不要求填主键
10. Ubuntu 24.04+ 需用 venv 安装 Python 依赖
