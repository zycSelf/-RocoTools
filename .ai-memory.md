## 核心数据模型

5 层数据：**属性(18) → 技能(469+) → 蛋组(15) → 精灵(466+) → 性格(30)**

| 数据 | UID 格式 | 存储 |
|------|----------|------|
| 属性 | `elem_{id}` | data/elements/ |
| 技能 | `skill_{N}` | data/skills/ |
| 蛋组 | 按名称 | data/eggs/ |
| 精灵 | `pet_{id}` 或 `pet_{id}_{N}` | data/pets/ |
| 性格 | 按 id (1-30) | data/natures/ |

精灵字段包含：element(属性引用)、sub_element(副属性)、egg_groups(蛋组名称列表)、thumb_url(缩略图)、detail(含技能/克制/立绘)

性格字段：name(名称)、stat_up(增加属性)、stat_down(减少属性)、sub_natures(5个子性格描述)

## 爬虫 (crawler/)

- 入口：`crawler/run.py --full` / `--update`
- 防限流：`utils/request.py` 提供 UA 池随机轮换、(2.0, 5.0)s 随机间隔、指数退避重试
- 下载器：`utils/downloader.py` 也加入随机 UA 和间隔抖动
- 执行顺序：属性 → 技能 → 蛋组 → **性格** → 精灵列表 → 精灵详情
- 增量模式：前置数据每次都重新拉取；精灵详情按 version 判断，但映射关系始终刷新
- 完成后自动：打印完整性汇总 + 同步 SQLite 数据库
- 每个爬虫自动生成 `*_report.md` 完整性校验报告

## 后台 (app/server/)

- Express + better-sqlite3
- Service 层独立（可复用到 Electron IPC）
- RESTful API：`/api/elements`、`/api/skills`、`/api/eggs`、`/api/pets`、`/api/natures`
- API 内存缓存：elements/eggs/natures 10分钟，skills/pets 5分钟
- 静态资源：`/public/...` 指向 `data/public/`
- 前端托管：构建产物在 `server/public/`，SPA fallback

### 数据同步流程
```bash
cd app/server && node sync_db.js
```
执行顺序：
1. 生成缩略图（128px WebP q60，需 sharp）+ 更新 pet_list.json 写入 thumb_url
2. 初始化数据库（建表）
3. 导入数据（JSON → SQLite，含性格表）

## 前端 (app/client/)

- Vue 3 + Vue Router + Vite（代码分割：Vue 独立 chunk）
- TailwindCSS + Sass
- base 路径：`/rocotools/`
- 构建输出到 `app/server/public/`（生产模式单服务部署）
- 开发模式 :5173 代理 API 到 :3000
- 字体：MIANFEIZITI（洛克专用免费字体，index.html 预加载）+ 系统字体（PingFang SC/微软雅黑）
- 主题色：`#D69F23`（primary-500）
- 暗色模式：`body.dark` 类切换
- SEO：index.html 含 description/keywords/Open Graph meta 标签
- 图片懒加载：IntersectionObserver + 并发队列（max 6 张同时加载）
- API 请求：失败自动重试（2次/500ms 间隔）
- 响应式：三端适配 sm(640px)/md(768px)/lg(1024px)

## 部署架构

- 服务器：腾讯云轻量 4核4G/3M/40G SSD，Ubuntu
- 域名：eachz.cn（备案已通过），base: /rocotools/
- 进程守护：PM2（pm2 reload 零停机更新）
- Nginx：HTTP/2 + Gzip + SSL（Let's Encrypt）+ OCSP + 安全头
- 反向代理：proxy_http_version 1.1 连接复用
- 静态资源缓存：Vite assets 1年 immutable，图片 90 天
- 部署方式：`./deploy.sh`（git pull + npm install + build + pm2 reload + nginx reload）
- 定时部署：crontab 每天 00:00 自动执行 deploy.sh
- 缩略图：列表页 202MB 原图 → 2.5MB WebP 缩略图（99% 压缩）

## 性能优化

| 优化项 | 说明 |
|--------|------|
| HTTP/2 | 多路复用，消除并发连接限制 |
| WebP 自动返回 | Nginx 检测浏览器支持，透明返回 WebP |
| 图片懒加载 | IntersectionObserver + 并发队列（max 6） |
| API 缓存 | 内存缓存 5-10 分钟（X-Cache 头） |
| Gzip 压缩 | 文本资源全量压缩，gzip_min_length 256 |
| 长缓存 | Vite 带 hash 的 assets 缓存 1 年 immutable |
| 代码分割 | Vue 框架独立 chunk |
| 系统字体 | 不加载 Google Fonts，零额外请求 |
| keepalive 300s | 减少切页签后首次刷新失败 |
| API 自动重试 | 失败后重试 2 次/500ms 间隔 |
| open_file_cache | Nginx 缓存文件句柄减少磁盘 IO |

## 属性克制计算规则

**不从 BWIKI 页面爬取**，通过 `element_chart.json` + 精灵属性**实时计算**。

### 计算方法（双属性乘积法）

对于攻击属性 A 打防御方（主属性 E1 + 副属性 E2）：
1. 分别计算 A 对 E1 的倍率和 A 对 E2 的倍率
2. 单属性倍率判定：
   - A 的 `strong_against` 包含 Ex → ×2（strong）
   - Ex 的 `resistant_to` 包含 A → ×0.5（resist）
   - 否则 → ×1
3. 两个倍率相乘得到最终倍率
4. 特殊规则：双重克制(2×2) → `double_strong`(×3)；双重抵抗(0.5×0.5) → `double_resist`(×0.25)

### 倍率常量
- strong: 2.0 | resist: 0.5 | double_strong: 3.0 | double_resist: 0.25

## 数据规范

- 空值使用 `null`，不用空字符串；空数组用 `[]`
- 图片路径统一 `/public/...` 格式
- UID 格式：`elem_{id}`、`skill_{N}`、`pet_{id}` 或 `pet_{id}_{N}`
- 增量判断通过 `version` 字段
- 执行顺序严格按依赖关系：属性→技能→蛋组→性格→精灵列表→精灵详情
- 详细字段定义见 `data/FIELDS.md`
- 结构化规则见 `data/STRUCTURE_RULES.md`

## 重要约定

1. `data/` 目录不纳入 git（含大量图片和生成数据）
2. `DEPLOY.md` 不入 git（含部署信息）
3. `nginx.conf` 和 `deploy.sh` 已纳入 git
4. 爬虫完成后自动同步 SQLite（需 server 已 npm install）
5. 增量模式下即使详情不重新爬取，也会刷新所有映射关系
6. 前端 build 产物输出到 `app/server/public/`，server 自动托管
7. Service 层不依赖 Express，后续可无缝迁移到 Electron IPC
8. 技能详情 learners 按 pet_id + skill_type 去重
9. import.js 导入前清空 pet_skills 和 variants_map 防止重复
10. Ubuntu 24.04+ 需用 venv 安装 Python 依赖（PEP 668）
