## 核心数据模型

4 层关联：**属性(18) → 技能(469+) → 蛋组(15) → 精灵(466+)**

| 数据 | UID 格式 | 存储 |
|------|----------|------|
| 属性 | `elem_{id}` | data/elements/ |
| 技能 | `skill_{N}` | data/skills/ |
| 蛋组 | 按名称 | data/eggs/ |
| 精灵 | `pet_{id}` 或 `pet_{id}_{N}` | data/pets/ |

精灵字段包含：element(属性引用)、egg_groups(蛋组名称列表)、detail(含技能/克制/立绘)

## 爬虫 (crawler/)

- 入口：`crawler/run.py --full` / `--update`
- 并发：详情页 5 线程(0.5s)，图片 10 线程(0.1s)
- 限流：567/429 自动重试(30s×次数，最多3次)
- 执行顺序：属性 → 技能 → 蛋组 → 精灵列表 → 精灵详情
- 增量模式：前置数据每次都重新拉取；精灵详情按 version 判断，但映射关系(egg_groups/element/skill_ref)始终刷新
- 完成后自动：打印完整性汇总 + 同步 SQLite 数据库

## 属性克制计算规则

**不从 BWIKI 页面爬取**（页面模板不统一），通过 `element_chart.json` + 精灵属性**实时计算**。

### 计算方法（双属性乘积法）

对于攻击属性 A 打防御方（主属性 E1 + 副属性 E2）：
1. 分别计算 A 对 E1 的倍率和 A 对 E2 的倍率
2. 单属性倍率判定：
   - A 的 `strong_against` 包含 Ex → ×2（strong）
   - Ex 的 `resistant_to` 包含 A → ×0.5（resist）
   - 否则 → ×1
3. 两个倍率相乘得到最终倍率
4. 特殊规则：双重克制(2×2) → `double_strong`(×3)；双重抵抗(0.5×0.5) → `double_resist`(×0.25)

### 倍率常量
- strong: 2.0 | resist: 0.5 | double_strong: 3.0 | double_resist: 0.25

## 打击面分析逻辑

### PetDetail 内嵌（CoverageAnalysis.vue）
- **理论最大模式**：贪心算法选N个属性覆盖最多属性，可切换含/不含血脉、物攻/魔攻、槽位数
- **自定义组合模式**：手动选属性+血脉，实时计算覆盖
- 血脉约束：每精灵只能选一种血脉

### 独立页面（Coverage.vue）
- 4格选属性 → 实时查询匹配精灵
- 后台 `findByCoverage`：按 pet_uid 精确匹配形态，按 pet_id 去重
- 结果分两组：自学(normal) / 需血脉(bloodline，按血脉属性分组)
- 跳转携带 query 参数（coverage/bloodline），PetDetail 只消费一次

## 移动端适配规范

详见 `app/client/RESPONSIVE.md`，核心要点：

- **断点**：`md:` (768px) 为主断点，mobile-first 写法
- **导航**：移动端汉堡菜单，桌面端水平导航+下拉
- **表格**：移动端用卡片替代（技能列表）或横滚（克制表）
- **字号**：body `text-sm md:text-base`，标题 `text-xl md:text-2xl`
- **间距**：card `p-3 md:p-5`，内容区 `px-3 md:px-8`
- **图标**：属性 `w-6 md:w-8`，立绘 `w-20 md:w-28`（PetCard）
- **雷达图**：动态尺寸 160/200（通过 window.innerWidth 监听）
- **文本截断**：移动端 `line-clamp-2`，桌面端不限

## 数据规范

- 空值使用 `null`，不用空字符串；空数组用 `[]`
- 图片路径统一 `/public/...` 格式
- UID 格式：`elem_{id}`、`skill_{N}`、`pet_{id}` 或 `pet_{id}_{N}`
- 增量判断通过 `version` 字段
- 执行顺序严格按依赖关系：属性→技能→蛋组→精灵列表→精灵详情
- 详细字段定义见 `data/FIELDS.md`
- 结构化规则见 `data/STRUCTURE_RULES.md`

## 重要约定
