## 核心数据模型

7 层数据：**属性(18) → 技能(469+) → 蛋组(15) → 精灵(466+) → 性格(30) → 赛季 → 活动日历**

附加数据：**皮卡月刊**（角色时装，多精灵绑定、单概念图）

| 数据 | UID/PK | 存储 |
|------|--------|------|
| 属性 | `elem_{id}` | data/elements/ |
| 技能 | `skill_{N}` | data/skills/ |
| 蛋组 | 按名称 | data/eggs/ |
| 精灵 | `pet_{id}` 或 `pet_{id}_{N}` | data/pets/ |
| 性格 | id (1-30) | data/natures/ |
| 赛季 | S1/S2... | 管理端配置（仅数据库） |
| 活动 | 自增 id | season_events 表（管理端配置） |
| 皮卡月刊 | 自增 id | pika_monthlies + pika_monthly_pets 表（管理端配置） |

## 前端全局功能

### 图片预览系统

图片预览功能已全局化，所有管理端页面共享同一个预览组件：

**架构**：
- `src/composables/useImagePreview.js`：模块级单例，提供全局预览状态
- `src/components/shared/ImagePreview.vue`：预览组件，通过 Teleport 挂载到 body
- 在 `App.vue` 中全局挂载，所有管理端路由共享

**使用方式**：
```js
import { useImagePreview } from '@/composables/useImagePreview'
const { openPreview } = useImagePreview()
// 在图片点击事件中调用
openPreview(imageUrl)
```

**关键实现细节**：
1. Teleport 到 body 的组件必须使用非 scoped 样式，否则样式失效
2. 小尺寸图片（如 5x5 图标）需用透明 div 包裹扩大点击区域
3. 点击遮罩层可关闭预览
4. `v-click-outside` 指令在 `main.js` 中全局注册

**扩展**：后续所有需要图片预览的页面，只需导入 `useImagePreview` 并调用 `openPreview()` 即可

## 爬虫 (crawler/)

- 入口：`crawler/run.py --full` / `--update`
- 防限流：UA 池 + 随机间隔 + 指数退避重试
- 执行顺序：属性 → 技能 → 蛋组 → 性格 → 精灵列表 → 精灵详情
- 完成后自动：打印完整性汇总 + 同步 SQLite 数据库
- 每个爬虫自动生成 `*_report.md` 完整性校验报告

## 后台 (app/server/)

- Express + better-sqlite3 + dotenv
- RESTful API：elements/skills/eggs/pets/natures/seasons/events
- 统计接口：`/api/stats`（首页概览，返回 pets/skills/elements/eggs/natures 数量）
- 皮卡月刊 API：`/api/pika-monthlies`（列表含关联精灵）
- 管理端 API：`/api/admin/*`（JWT 鉴权）
- API 内存缓存：elements/eggs/natures/seasons 10分钟，skills/pets/events 5分钟
- 静态资源：`/public/` → data/public/，`/uploads/` → data/uploads/
- 前端托管：server/public/，SPA fallback
- 确保 data 目录存在：`fs.mkdirSync` 在启动时自动创建

### 数据同步 sync_db.js
1. 生成缩略图（128px WebP q60）→ 更新 pet_list.json
2. 生成 WebP 副本（全部图片）
3. 建表（CREATE IF NOT EXISTS，不删库）+ **自动迁移缺失列**（migrateColumns）
4. 导入数据（跳过 manual_edit=1 的记录，冲突存入 pending_conflicts.json）
5. 迁移 show_shiny 列（默认值1）
6. 规范化身高体重数据
7. 清洗技能等级字段
8. 同步进化链（多路线合并）
9. 同步最终形态标记
10. 同步默认图鉴课题

### 远程数据同步 (scripts/sync_from_server.sh)

从服务器拉取数据库、图片、赛季备份到本地开发环境。

**前置条件**：
1. 配置 `scripts/.env`（从 `scripts/.env.example` 复制），填入 `REMOTE_USER` / `REMOTE_HOST` / `REMOTE_PROJECT`
2. 配置 SSH 免密登录

**常用命令**：
```bash
bash scripts/sync_from_server.sh --db        # 拉取服务器DB到本地（自动备份+完整性校验）
bash scripts/sync_from_server.sh --images    # 增量同步图片
bash scripts/sync_from_server.sh --seasons   # 同步赛季备份到 temp/seasons/
bash scripts/sync_from_server.sh --all       # 全部同步
bash scripts/sync_from_server.sh --all --full  # 全量同步（忽略时间戳）
```

**同步机制**：有 rsync 用 rsync 增量；无 rsync（Windows Git Bash）自动 fallback 到 scp+tar 差异同步。每次成功同步后记录时间戳到 `scripts/.last_image_sync`。

### 数据库迁移
- init.js 的 `migrateColumns()` 自动解析 schema.sql，对比实际表结构，ALTER TABLE ADD COLUMN 补充缺失列
- 无需删库重建，新增字段只需改 schema.sql 后跑 sync_db.js

### 数据保护
- `manual_edit` 字段：管理端编辑自动标记为 1
- import.js 跳过标记记录，生成冲突报告
- 管理端「数据审查」页面逐条对比处理

### 导航标签系统

用户端顶部导航栏通过数据库驱动，支持父子结构（二级下拉）：

| 字段 | 说明 |
|------|------|
| `tab_key` | 唯一标识键（创建后不可修改） |
| `label` | 显示名称 |
| `route` | 路由路径（父级标签可为空） |
| `parent_key` | 父级标签的 tab_key（为空则是顶级标签） |
| `is_visible` | 是否显示 |
| `sort_order` | 排序权重（数字越大越靠前） |

**父子结构逻辑**：
- 子标签的 `parent_key` 指向父级标签的 `tab_key`
- 父级标签 `route` 可为空（仅作为下拉菜单）
- 桌面端：鼠标悬停父级标签显示子标签下拉
- 移动端：点击父级标签展开/折叠子标签

**管理端配置**：
- 访问 `/admin/nav-tabs` 管理导航标签
- 勾选「作为父级（下拉菜单，不跳转）」可创建无路由的父级标签
- 编辑子标签时，在「父级标签」下拉框选择对应父级

## 管理端

- 鉴权：JWT（4h有效），密码/secret 通过 .env 配置
- 通用 CRUD：EDITABLE_TABLES 配置表名/主键/可编辑字段/autoIncrement
- 图片上传：IMAGE_TYPES 配置目录/后缀/DB映射，isUpload 类型存到 data/uploads/
- 素材库：`data/uploads/library/`，接口 `GET/POST /api/admin/library`、`DELETE /api/admin/library/:filename`；前端 `adminApi.libraryList/libraryUpload/libraryDelete`
- 统一素材管理：`/admin/media` 页面，按业务分类浏览所有图片（素材库/精灵/皮卡/赛季/活动/技能/属性），支持网格/列表视图、搜索、多选批量删除
- 素材管理接口：`GET /api/admin/media`（列出所有图片）、`DELETE /api/admin/media`（按路径删除）、`POST /api/admin/media/copy-to-business`（从素材库复制到业务目录）
- 通用上传组件：`components/shared/ImageUploader.vue`，props: `uploadType`/`uploadUid`/`btnClass`/`uploadLabel`，emit: `uploaded(path)`
  - 两种方式：本地直传（到业务目录）+ 素材库选取（支持「复制到业务目录」选项）
- 备份系统：
  - 临时备份（backups/）：1次确认删除
  - 赛季备份（backups/seasons/）：3次确认删除
  - 恢复前快照（backups/snapshots/）：可命名
- 赛季管理：通行证(2)+传说(多只，JSON数组)+限定(8)+异色(8)，PetPicker 选取
- 活动管理：版本活动(banner+单段日期)+常驻课题(多段日期)，支持图片上传
- 皮卡月刊管理：期刊配置 + 精灵关联 + 概念图/时装图上传 + 同步活动
- 通用 CRUD create API 返回 lastInsertRowid（供图片上传关联用）

### 赛季公告系统

赛季详情页和首页均支持 Markdown 公告展示，通过管理端赛季配置页面编辑。

**公告字段**：
- 赛季详情公告：`announcement_title` + `announcement_content`
- 首页公告：`home_announcement_title` + `home_announcement_content`

**自定义 Markdown 语法**：

| 语法 | 渲染效果 |
|------|----------|
| `![pet:uid]` | 精灵头像图标（24px圆形） |
| `![skill:uid]` | 技能图标（20px） |
| `![ability:path]` | 特性图标（2em，与行高齐平） |
| `![element:path]` | 属性图标（1.2em，用于技能属性展示） |
| `![img:path]` | 内联图片（56px固定尺寸） |
| `![shiny:uid]` | 异色立绘（56px），加载失败自动隐藏 |

**多传说精灵**：`seasons.legend_pet` 支持 JSON 数组格式（如 `["pet_001","pet_002"]`），兼容旧单值格式。

**公告生成脚本**：
- `scripts/generate_launch_notes.js`：开服公告（读取单个DB快照）
- `scripts/generate_patch_notes.js`：更新公告（对比两个DB快照差异）
- 两个脚本均输出含自定义语法的 Markdown，由前端动态渲染
- 所有列表（精灵/技能/赛季奇遇等）按编号自然排序（从 uid 提取数字）
- 新增技能表格包含属性图标（`![element:path]`）和类型列

## 重要约定

### 前端 API 导入规范

- `@/api` → `src/api/index.js`（用户端 API：elementsApi/skillsApi/eggsApi/petsApi/naturesApi/seasonsApi/eventsApi）
- `@/api/admin` → `src/api/admin.js`（管理端 API：adminApi）
- **禁止**在 `api/index.js` 中使用但未添加重导出。新建页面时，若从 `@/api` 导入了 `adminApi`，必须在 `api/index.js` 顶部添加 `export { adminApi } from './admin.js'`

### 前端页面开发规范

- 新建管理端页面需在三处添加入口：
  1. `App.vue` 桌面导航栏（顶栏）
  2. `App.vue` 移动端导航（汉堡菜单）
  3. `AdminDashboard.vue` 的 `navCards` 数组（Dashboard 卡片）
- 路由配置在 `router/index.js`

### Vue 组合式 API 导入规范

- 使用生命周期钩子时必须完整导入：`import { ref, computed, onMounted, onUnmounted } from 'vue'`
- **禁止**只导入 `onMounted` 而使用 `onUnmounted`（或其他组合）
- 需要清理的副作用（如事件监听）必须使用 `onUnmounted` 清理

### Vue 模板属性语法规范

- **禁止**在模板属性中使用反引号模板字符串，如 `:uid="\`${a}_${b}\`"`
- 必须改用字符串拼接：`:uid="a + '_' + b"`，或提取为 `computed`
- 原因：`vue-eslint-parser` 无法正确解析模板属性中的反引号，会误报 `<script setup>` 位置错误

### Vue 自定义指令注册规范

- 全局指令在 `main.js` 中通过 `app.directive('指令名', 指令定义)` 注册
- 指令定义包含 `mounted(el, binding)` 和 `unmounted(el)` 生命周期
- 示例：`v-click-outside` 指令用于检测点击元素外部的事件
- 指令定义可以放在 `composables/` 目录下（如 `useLazyImage.js`）

### 响应式设计断点规范

- **Mobile-first**：从移动到桌面渐进增强
- **断点**：
  - `sm:` (640px) — 手机 → 平板过渡
  - `md:` (768px) — 导航栏切换为桌面版
  - `lg:` (1024px) — 平板 → 桌面过渡
- **适配策略**：
  - 表格在小屏使用卡片布局或横向滚动
  - 导航栏：桌面水平导航，移动端汉堡菜单
  - 图片：使用懒加载 + 响应式尺寸

### Git 忽略规则

1. `data/` 不入 git（含图片和生成数据）
2. `data/uploads/` 不入 git（手动上传图片，不被爬虫覆盖）
3. `deploy.sh`、`setup-nginx.sh`、`DEPLOY.md`、`.env` 不入 git
4. `nginx.conf` 入 git 但用占位符（`<YOUR_DOMAIN>`/`<PROJECT_DIR>`）
5. `app/server/data/` 不入 git（数据库文件）
6. `app/server/public/` 不入 git（前端构建产物）
7. `node_modules/` 不入 git
8. `.vscode/settings.json` 入 git（配置默认终端为 Git Bash）

### 数据库迁移规范

5. init.js 不删除旧数据库（保留 seasons 等手动录入数据）+ 自动迁移缺失列
6. 管理端编辑自动标记 manual_edit=1，import 时跳过
7. 赛季/活动数据仅在管理端配置，不通过爬虫获取
8. 活动 API 默认只返回当前活跃的（end_date >= today），管理端传 all=1 获取全部
9. 通用 CRUD 的 autoIncrement 表创建时不要求填主键
10. Ubuntu 24.04+ 需用 venv 安装 Python 依赖
11. 修改数据库表结构：只需修改 `schema.sql`，然后运行 `sync_db.js` 自动迁移

### 数据库路径规范（强制）

**统一入口：`app/server/src/db/connection.js`**

| 导出 | 说明 |
|------|------|
| `DB_PATH` | `app/server/data/roco.db` |
| `DATA_DIR` | 项目根目录 `data/`（爬虫数据 + 图片） |
| `getDb()` | 共享只读连接（services/routes 用，不 close） |
| `getWriteDb()` | 可写连接（管理端/脚本用，需 close） |

- **禁止**在任何文件中自行 `path.join(__dirname, ...)` 拼接数据库路径
- services 层：`const { getDb } = require('../db/connection'); const db = getDb();`
- 路由层（只读）：`const { getDb } = require('../db/connection');`
- 路由层（写入）：`const { getWriteDb } = require('../db/connection');`
- 独立脚本（init/import）：`const { DB_PATH, DATA_DIR } = require('./connection');`
